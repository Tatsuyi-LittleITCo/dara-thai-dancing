## Critical Issues

### Issue 1: Severely Outdated Dependencies

**Severity:** Critical
**Location:** `package.json`
**Category:** Security / Dependencies

**Problem:**
- `next` 14.2.5 → latest is **15.3.3** (2 major versions behind)
- `react` / `react-dom` 18.2.0 → latest is **19.1.0** (1 major version behind)
- No `package-lock.json` exists — builds are non-reproducible and `npm audit` cannot run

**Impact:**
Missing security patches, no CVE scanning possible, potential deployment failures from non-deterministic installs.

**Recommendation:**
Upgrade to Next.js 15 + React 19. Generate a lockfile.

```bash
# Step 1: Generate lockfile for current versions first
npm i --package-lock-only

# Step 2: Upgrade
npm install next@latest react@latest react-dom@latest

# Step 3: Audit
npm audit
```

Note: Next.js 15 has breaking changes (async params in pages, new image component defaults). Test thoroughly after upgrade.

### Issue 2: Broken JSX in FacebookEmbed

**Severity:** Critical
**Location:** `components/FacebookEmbed.jsx:16-18`
**Category:** Code

**Problem:**
The blockquote contains broken HTML — there's an `</a>` closing tag with no opening `<a>` tag:

```jsx
<blockquote cite={pageUrl} className="fb-xfbml-parse-ignore">
  https://www.facebook.com/darathaidancing
    Dara Thai Dancing School
  </a>   // ← no matching <a> tag
</blockquote>
```

**Impact:**
Invalid DOM, React hydration warnings, potential render failures.

**Recommendation:**
```jsx
<blockquote cite={pageUrl} className="fb-xfbml-parse-ignore">
  <a href={pageUrl}>Dara Thai Dancing School</a>
</blockquote>
```

### Issue 3: No Mobile Navigation

**Severity:** Critical
**Location:** `globals.css:34`, `components/Header.jsx`
**Category:** Usability

**Problem:**
At `max-width: 1200px`, the nav is hidden with `display: none` and there is no hamburger menu or alternative. Mobile users have **no way to navigate** between pages.

**Impact:**
The site is completely un-navigable on mobile/tablet devices.

**Recommendation:**
Add a hamburger menu toggle for mobile. This requires:
1. A menu button visible only on small screens
2. A slide-out or dropdown nav panel
3. Focus trapping and aria attributes for accessibility

### Issue 4: Missing `<head>` Essentials

**Severity:** Critical
**Location:** `app/layout.jsx`
**Category:** SEO / Performance

**Problem:**
No viewport meta tag, no charset declaration, no Open Graph tags, no favicon link. The `<html>` tag has no `lang` attribute that updates with the language toggle (always `en`).

**Impact:**
- No viewport = broken mobile rendering
- No OG tags = poor social sharing
- Static `lang="en"` when Thai is selected = accessibility/SEO issue

**Recommendation:**
Add viewport and charset via Next.js metadata API. Set `lang` dynamically based on language context.

## Major Issues

### Issue 5: CSS Duplication and Conflicts

**Severity:** Major
**Location:** `app/globals.css`
**Category:** Code Quality

**Problem:**
Multiple conflicting declarations:
- `.btn` defined twice (lines 26-28 and 100-109) with different backgrounds
- `.layout-with-aside` defined twice (lines 122-127 and 178-184) with different column templates
- `.aside-sticky` defined three times with different `top` values
- `.card-full` defined twice
- `!important` scattered throughout

**Impact:**
Unpredictable styling, hard to maintain, cascade conflicts.

**Recommendation:**
Consolidate all duplicate rules. Consider adopting CSS Modules or Tailwind CSS for scoped, conflict-free styles.

---

### Issue 6: Excessive Inline Styles

**Severity:** Major
**Location:** All page components, `Header.jsx`, `layout.jsx`
**Category:** Code Quality

**Problem:**
Nearly every component uses inline `style={{}}` objects extensively. Examples:
- `Header.jsx`: 5 inline style objects
- `page.jsx` (home): 8+ inline style objects
- `about/page.jsx`: 6 inline style objects

**Impact:**
Styles are not reusable, not responsive (no media queries in inline styles), hard to maintain, and create new objects on every render.

**Recommendation:**
Move all styles to CSS classes (CSS Modules or globals). Inline styles should be reserved for truly dynamic values only.

### Issue 7: No Image Optimization

**Severity:** Major
**Location:** `about/page.jsx:49`, `Header.jsx:32`
**Category:** Performance

**Problem:**
Using raw `<img>` tags instead of Next.js `<Image>` component. Images are served unoptimized (no WebP/AVIF, no srcset, no lazy loading on the logo).

**Impact:**
Larger page weight, slower LCP, poor Core Web Vitals.

**Recommendation:**
Use `next/image` with proper `width`/`height` or `fill` props. Note: with `output: "export"`, you'll need a custom image loader or use `unoptimized: true` in next.config.js (still gets proper `<img>` attributes).

### Issue 9: Facebook SDK Not Loaded

**Severity:** Major
**Location:** `components/FacebookEmbed.jsx`, `app/layout.jsx`
**Category:** Code

**Problem:**
The `FacebookEmbed` component renders `data-*` attributes for the Facebook SDK (`fb-page`, `data-href`, etc.) but the Facebook SDK script (`sdk.js`) is never loaded anywhere in the app.

**Impact:**
The Facebook embed renders as a plain blockquote fallback (which is also broken — see Issue 2). The Facebook section is non-functional.

**Recommendation:**
Either load the Facebook SDK via `next/script`, or switch to an iframe-based embed that doesn't require the SDK.

### Issue 10: HTML in Translation Strings

**Severity:** Major
**Location:** `components/LangProvider.jsx:37`
**Category:** Code / Security

**Problem:**
The `mission_body` translation contains raw HTML (`<div><b>krur ploy</b></div>`). This is rendered as literal text by React (not parsed as HTML), so it displays the tags visually to users.

**Impact:**
Users see raw HTML tags on the About page. If `dangerouslySetInnerHTML` were used to "fix" this, it would create an XSS vector.

**Recommendation:**
Use React elements or a simple markup approach:
```jsx
// In the component, split the translation or use a render function
<p>...created by <strong>Krur Ploy</strong> as a way to...</p>
```

## Suggestions

### Suggestion 1: Adopt Tailwind CSS

The site has ~230 lines of globals.css with significant duplication and specificity issues. Tailwind CSS would eliminate all these problems and provide a consistent design system out of the box. Next.js has first-class Tailwind support.

### Suggestion 2: Add SEO Metadata per Page

Each page exports static content but has no per-page metadata (title, description, OG image). Use Next.js `metadata` exports in each page for proper SEO.

### Suggestion 3: Add a Sitemap Generator

`robots.txt` references `/sitemap.xml` but no sitemap is generated. Use `next-sitemap` or Next.js built-in sitemap generation.
